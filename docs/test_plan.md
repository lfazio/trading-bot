## 1. Introduction

### 1.1 Purpose
Define how every approved requirement (SRS / SDS / SDD — 217 items) is
verified before the system is allowed to trade. Each test case is mapped
explicitly to one or more requirements, and the gate condition for opening
Phase 5 is `reached TP: 100%` from `tools/traceability.py --report`.

### 1.2 Scope
Covers unit, integration, property, and end-to-end tests; backtesting
validation harnesses; risk-engine drills; tax-engine boundary suites;
kill-switch trip / recovery drills; structured-product stress drills;
walk-forward / out-of-sample evaluation; broker-adapter conformance;
performance / determinism gates; CI configuration.

Out of scope: production monitoring, capacity planning beyond the
performance gate (REQ_SDD_PER_004), and live-trading manual operator
runbooks (those are Phase 7 outputs).

### 1.3 Verification methods
Inherited from SRS §1.3:
- **T** — Test (automated; pytest / hypothesis / pytest-bdd)
- **A** — Analysis (proof, formal calculation, derivation review)
- **I** — Inspection (lint, type-check, code review, doc review)
- **D** — Demonstration (end-to-end run, drill)

### 1.4 Notation
- `TC_<MOD>_<NUM>` — test case (specifies *what* to test); not a REQ id.
- `REQ_TP_*` — test-plan-level requirements (own the *test design*).
- "Verifies" — the REQs a test case demonstrates compliance with.

## 2. Test Strategy

### 2.1 Test pyramid
| Tier | Tooling | Target % of suite | Run time budget |
|---|---|---:|---|
| Unit (pure-function) | `pytest`, `hypothesis` (property tests) | ≥ 70% | < 30 s |
| Integration (multi-module, mock adapters) | `pytest`, `pytest-bdd` | ≈ 25% | < 5 min |
| End-to-end (mock broker + mock data, full loop) | `pytest`, scripted scenarios | ≈ 5% | < 10 min |
| Performance / property gates | `pytest-benchmark`, `hypothesis` | as needed | < 2 min |

### 2.2 Tools
- `pytest` (test runner, fixtures, parametrization)
- `hypothesis` (property-based; tax boundaries, phase hysteresis,
  decimal arithmetic, walk-forward windowing)
- `pytest-bdd` (KS triggers and operator-recovery scenarios written as
  Given/When/Then)
- `pytest-benchmark` (REQ_SDD_PER gates)
- `coverage.py` (branch coverage; gate at ≥ 90% on engine modules)
- `mypy --strict` (type checking)
- `ruff` (linting + naming conventions per REQ_SDD_NAM_*)
- Custom: `tools/check_imports.py` (acyclic-imports gate per REQ_SDD_IMP_003)
- Custom: `tools/traceability.py --check` (CI gate)

### 2.3 Test environment & determinism
- All randomness seeded; default seed `0xCAFE` for reproducibility.
- `EventClock` used in every test except those explicitly testing
  wall-clock behavior.
- `MockBrokerAdapter` and `MockMarketDataProvider` are the default
  adapters in tests; concrete adapters (XTB) run in a separate
  conformance suite gated by environment variable.
- Synthetic market series generated by a single fixture (`mock_market`)
  parameterized over `(seed, regime, length)` — produces identical bar
  series for identical tuples (REQ_SDD_TST_002).

### 2.4 Test data
- Bundled CSV fixtures for fundamental data covering ≥ 50 EU equities
  spanning all sectors required by the screener REQs.
- Synthetic regime stitching for stress runs: bull (2003–07), bear
  (2008–09), sideways (2015–16), high-vol (2020 Q1).
- Capital-injection schedules: empty, monthly DCA, lump-sum at month 6.
- Turbo universe fixture: 30 instruments per direction with varying
  knockout distance, leverage, and spread.

### 2.5 Coverage targets
- Branch coverage ≥ 90% on every engine module (`tax/`, `risk/`,
  `phase_engine/`, `screener/`, `turbo_selector/`, `structured_products/`,
  `capital_flow/`, `safety/`, `milestone_controller/`).
- Branch coverage ≥ 80% on adapters and `backtesting/`.
- Branch coverage ≥ 70% on `analytics/`, `dashboard/`, `main.py`.

### 2.6 CI gates
A change merges only when ALL of the following hold:
1. `ruff check` clean.
2. `mypy --strict` clean.
3. `pytest` 100% pass.
4. Coverage thresholds met (§2.5).
5. `tools/traceability.py --check` exits 0 (no drift; no warnings).
6. `tools/check_imports.py` clean (no cycles, no forbidden imports).
7. Performance benchmarks within budgets (REQ_SDD_PER_*).
8. Adapter conformance suite passes for every shipped concrete adapter.

## 4. Test-Plan Requirements (`REQ_TP_*`)

These design-level requirements own the *test design* itself. The Phase 6
test code SHALL satisfy them — they are not redundant with `REQ_SDD_TST_*`
(which constrains what the code under test must permit).

### 4.1 Test strategy — `REQ_TP_STR`

- **REQ_TP_STR_001** — The test suite SHALL be organized in three tiers (unit, integration, e2e) with the budget split documented in §2.1, and CI SHALL fail on tier budget regressions of more than 25%. *Derives from: REQ_NF_LIF_001.* *V: I, T*
- **REQ_TP_STR_002** — Property-based tests (`hypothesis`) SHALL be used for: tax boundaries, phase hysteresis, risk-parity weight invariants, deterministic mock-data generation, and walk-forward window arithmetic. *Derives from: REQ_NF_DET_001, REQ_SDD_TST_002.* *V: I, T*
- **REQ_TP_STR_003** — Kill-switch and recovery scenarios SHALL be specified as Given/When/Then BDD scenarios (`pytest-bdd`) so operator runbooks remain consistent with executable tests. *Derives from: REQ_S_KS_007, REQ_S_KS_009, REQ_NF_AUD_001.* *V: I, T*
- **REQ_TP_STR_004** — Concrete `BrokerAdapter` implementations SHALL pass the same conformance suite as `MockBrokerAdapter`; new adapters MAY ship only after passing this suite. *Derives from: REQ_F_BRK_002, REQ_F_BRK_003, REQ_SDD_TST_001.* *V: T*

### 4.2 Coverage & gates — `REQ_TP_COV`

- **REQ_TP_COV_001** — Branch coverage SHALL meet §2.5 thresholds; CI SHALL block merges below threshold. *Derives from: REQ_NF_TRC_001.* *V: T*
- **REQ_TP_COV_002** — `tools/traceability.py --check` SHALL exit 0 in CI; any drift, unknown REQ reference, or duplicate definition SHALL block the merge. *Derives from: REQ_NF_TRC_001, REQ_NF_LIF_002.* *V: T*
- **REQ_TP_COV_003** — Every approved SRS / SDS / SDD requirement SHALL be referenced by at least one test case in this plan; the gate condition for opening Phase 5 is `reached TP: 100%`. *Derives from: REQ_NF_TRC_001.* *V: T*
- **REQ_TP_COV_004** — Phase 6 test execution SHALL achieve `reached TEST: 100%` before any live or paper-trading deployment. *Derives from: REQ_NF_LIF_001, REQ_NF_TRC_001.* *V: T*

### 4.3 Fixtures & determinism — `REQ_TP_FIX`

- **REQ_TP_FIX_001** — All tests SHALL use `EventClock` and seeded RNG by default; tests using `WallClock` SHALL be marked `@pytest.mark.wallclock` and excluded from the default CI run. *Derives from: REQ_SDS_ARC_005, REQ_SDS_ARC_006.* *V: I, T*
- **REQ_TP_FIX_002** — Fundamentals fixtures SHALL cover ≥ 50 EU equities spanning the sectors and yield bands referenced by REQ_F_SCR_001. *Derives from: REQ_F_SCR_001.* *V: T*
- **REQ_TP_FIX_003** — Synthetic regime fixtures (BULL/BEAR/SIDEWAYS/HIGH_VOL) SHALL be reused across screener, strategies, backtesting, structured products, and meta-loop tests so regime semantics stay consistent across modules. *Derives from: REQ_F_MTO_008.* *V: I*
- **REQ_TP_FIX_004** — Capital-injection schedule fixtures SHALL include: empty, monthly DCA, lump-sum at month 6, and irregular ad-hoc; performance metrics SHALL be unchanged across schedules of equal cumulative inflow. *Derives from: REQ_F_CFL_002.* *V: T*

### 4.4 Performance & determinism gates — `REQ_TP_GAT`

- **REQ_TP_GAT_001** — `must_halt()` < 1 µs measured by `pytest-benchmark`; regression > 50% fails CI. *Derives from: REQ_SDD_PER_001.* *V: T*
- **REQ_TP_GAT_002** — Backtest throughput ≥ 10,000 ticks/s on a single core with the mock provider. *Derives from: REQ_SDD_PER_004.* *V: T*
- **REQ_TP_GAT_003** — Reproducibility: two runs of any shipped strategy with identical (seed, config_hash, data) tuples SHALL produce identical trade logs and equity curves; CI SHALL diff and fail on any difference. *Derives from: REQ_NF_REP_001, REQ_SDD_TST_006.* *V: T*

### 4.5 Test-of-the-test-plan — `REQ_TP_INT`

- **REQ_TP_INT_001** — A meta-test SHALL parse this document, extract every `TC_*` and its "Verifies" list, and assert that every approved REQ is named in at least one TC. *Derives from: REQ_NF_TRC_001.* *V: T*
- **REQ_TP_INT_002** — `tools/traceability.py --report` output SHALL be archived in CI as a build artifact for every commit. *Derives from: REQ_NF_LIF_001.* *V: T*
