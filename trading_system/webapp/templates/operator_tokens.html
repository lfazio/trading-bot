Operator-token management

CR-024 §7 — rotate the household secret + revoke per-account tokens by jti. Every mutation emits a LogCategory.SECURITY structured-log entry; the raw token is never logged (only its SHA-256 hash).

Rotate household secret

Household-token-gated. Rotation generates a fresh 64-byte random secret server-side + returns it ONCE. The previous secret enters the grace window — existing tokens signed with it keep verifying until the NEXT rotation drops it (REQ_F_TOK_003 rolling rotation).

Revoke an operator token

Per-account-token-gated. Add a (account_id, jti) row to the revocation list. After this call the token can no longer authenticate any operation — the revocation check runs BEFORE the TTL check.
